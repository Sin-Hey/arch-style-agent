# 极端测试用例

下面这些需求可以直接复制到系统输入框中，用于测试需求解析、架构推荐、评分矩阵、组合架构方案和重构建议是否稳定。

## 1. 空泛模糊需求

```text
我要开发一个管理系统，功能以后再说，希望好用、稳定、方便维护。
```

## 2. 极端高并发实时系统

```text
开发一个跨平台即时通讯系统，要求支持百万用户同时在线，消息必须秒级送达，支持群聊、离线消息、推送通知，后期还要扩展语音和视频通话。
```

## 3. 强一致企业系统

```text
开发一个银行核心账务系统，要求转账、扣款、对账必须强一致，不能出现重复扣款或数据丢失，实时性要求一般，但审计、权限和可靠性要求极高。
```

## 4. 高吞吐异步数据处理

```text
建设一个日志分析平台，每秒接收数十万条日志，需要清洗、过滤、聚合并分发到多个下游系统，允许最终一致，但要求高吞吐、可扩展和失败重试。
```

## 5. 低成本 Serverless 场景

```text
开发一个活动报名系统，平时访问量很低，但活动开始时会突然出现短时间高峰，希望运维成本低，可以自动扩缩容，核心功能是报名、通知和数据导出。
```

## 6. 插件扩展型系统

```text
开发一个代码编辑器平台，核心功能稳定，但需要支持第三方插件扩展语法高亮、格式化、调试器和主题，要求插件可以独立安装、卸载和升级。
```

## 7. 数据中心型系统

```text
开发一个科研数据共享平台，多个分析工具都要围绕同一份中心数据集进行读写和查询，要求数据集中管理、可追踪、权限控制清晰。
```

## 8. 读写分离高查询系统

```text
开发一个电商订单查询系统，写入订单时要保证可靠，查询订单列表和统计报表的请求非常多，读请求远高于写请求，希望读写模型可以分别优化。
```

## 9. 小团队快速交付

```text
一个 3 人团队要在 2 周内开发课程作业管理系统，包括登录、作业提交、教师批改和成绩统计，用户量不大，希望结构清晰、开发简单、方便演示。
```

## 10. 多团队大型平台

```text
开发一个大型在线教育平台，包括课程、直播、作业、支付、消息通知、学习记录和推荐系统，由多个团队并行开发，要求服务可以独立部署和扩展。
```

## 11. 严格低延迟控制系统

```text
开发一个工业设备监控与控制系统，需要毫秒级响应传感器数据，异常时立即触发控制指令，可靠性和实时性要求极高，数据分析可以异步完成。
```

## 12. 矛盾需求测试

```text
开发一个系统，既要求所有数据强一致、事务不能失败，又要求跨地域百万并发实时响应，还希望部署简单、成本很低、一个人能长期维护。
```
